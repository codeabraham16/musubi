El proyecto

¿Qué es Musubi?

Musubi (結び, "anudar" en japonés) es un servidor MCP que le da a los agentes de IA una memoria persistente que sobrevive entre sesiones. Es local-first, escrito en Go puro sobre un único archivo SQLite, y opera sin modelos (model-free): toda validación y búsqueda es determinista. El protocolo orquesta; el agente ejecuta.

🗃️ Memoria que se entiende
No solo guarda texto: detecta conflictos semánticos, consolida duplicados y se auto-repara — todo sin llamar a un LLM.
🔀 Orquesta trabajo
Motor de orquestación por DAG y pizarra compartida multi-agente para coordinar flujos de trabajo complejos.
🪙 Eficiente en tokens
Recall por presupuesto de tokens, calibración opt-in y memoria de código por gists para minimizar el footprint de contexto.
🚀 Listo para escalar
Migraciones versionadas, índice vectorial, retención automática y un modo servicio HTTP seguro y observable.
La disciplina

Invariantes innegociables

Los mismos principios desde el día 1 hasta hoy — no se aflojaron al crecer 24 versiones.

Go puro · sin CGo
Un solo binario por plataforma, sin dependencias nativas. Solo 3 deps directas.
Model-free
Ningún LLM en la validación o la búsqueda. Comportamiento determinista y auditable.
Local-first
Un único archivo SQLite. Tus datos no salen de tu máquina (salvo modo servicio explícito).
MCP orquesta
El servidor coordina y persiste; el agente ejecuta. Separación clara de responsabilidades.
Test 1:1
Casi tantas líneas de test como de código. La cobertura crece con cada feature.
Entrega encadenada
PRs encadenados, cada sub-track es un release menor, CI verde con -race.
